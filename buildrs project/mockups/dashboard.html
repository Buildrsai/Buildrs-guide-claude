Chapitre 01⏱ 5 min

Tu n'es pas un dev,
tu es un orchestrateur

Le plus grand blocage des non-développeurs n'est pas technique. C'est mental. Ce chapitre te donne la posture qui change tout.

Le mythe du développeur

Pendant des années, créer un logiciel = savoir coder. HTML, CSS, JavaScript, bases de données... Des années de formation. Cette époque est révolue.

Les modèles IA comme Claude écrivent, déboguent, refactorent et déploient du code — plus vite qu'un dev junior, souvent aussi bien qu'un senior.

Le chef d'orchestre

Un chef d'orchestre ne joue d'aucun instrument. Pourtant sans lui, l'orchestre joue faux. Son rôle : la vision, le tempo, la correction en temps réel.

Toi, c'est exactement ton rôle avec l'IA. Tu ne codes pas. Tu diriges. L'IA exécute.

Changement de paradigme

La compétence clé de 2026 n'est pas de coder. C'est de savoir donner des instructions précises à des agents IA pour qu'ils produisent exactement ce que tu veux.

Les 3 compétences de l'orchestrateur
  • Formuler clairement : décrire un problème, un contexte, un objectif avec précision
  • Évaluer la qualité : savoir si le résultat correspond à ce que tu voulais
  • Itérer vite : corriger, affiner, recommencer jusqu'au bon résultat
Ce que ce guide n'est pas

Ce guide ne t'apprend pas à coder. Il t'apprend à maîtriser Claude pour que Claude code à ta place — et fasse bien plus que ça.

  • 3

Chapitre 04⏱ 6 min

Les modèles Claude
Haiku · Sonnet · Opus

Claude n'est pas un seul modèle. C'est une famille avec des compromis différents. Choisir le bon modèle, c'est économiser de l'argent et du temps.

La famille de modèles
ModèleVitesseIntelligenceCoût APIUsage idéal

Sonnet⚡ Rapide★★★$$Usage quotidien, code, Claude Code

Sonnet — ton cheval de bataille

Claude Sonnet est le modèle par défaut de Claude Code et de claude.ai. Pour 95% de tes usages, Sonnet est le bon choix : code, analyse, rédaction, debug, stratégie business.

Conseil coût

Ne lance jamais Opus par défaut. Commence avec Sonnet. Si la réponse est insuffisante, passe à Opus. Tu économises 80% des coûts API.

Extended Thinking

Certains modèles proposent le mode Extended Thinking — Claude réfléchit "à voix haute" avant de répondre : teste des hypothèses, s'autocorrige, arrive à de meilleures conclusions.

À activer dans l'interface Claude.ai : bouton de configuration du modèle → Extended Thinking.

mémoire, interface

La plupart des gens n'utilisent que 20% des fonctionnalités de Claude.ai. Voici les 80% qui changent tout.

Les Projets — ta fonctionnalité killer

Un Projet = un espace de travail persistant. Tout ce que tu upload et configures est disponible dans chaque conversation du projet.

  • Instructions permanentes : tu définis le rôle de Claude, le ton, les règles, le contexte une seule fois

  • Conversations organisées : tout est regroupé, rien ne se perd
Exemple concret

Crée un projet "Mon Micro-SaaS". Upload ton brief produit et ta cible. Dans chaque conversation, Claude connaît déjà ton contexte. Zéro répétition.

La mémoire utilisateur

Claude peut se souvenir d'informations entre conversations. Il stocke automatiquement : ton prénom, tes projets en cours, tes préférences de format, tes habitudes de travail.

Pour voir et modifier : Paramètres → Mémoire. Tu peux ajouter, modifier et supprimer des entrées manuellement.

Les Artifacts

Quand Claude génère du code HTML, React, SVG ou un document, il l'affiche dans un artifact — un panneau latéral interactif. Tu vois le résultat visuel immédiatement.

Les artifacts sont modifiables en direct : "modifie la couleur du bouton en rouge" → l'artifact se met à jour sans copier-coller.

Chapitre 07⏱ 10 min

CLAUDE.md —
ton chef d'orchestre

Le fichier le plus important de tout ton projet. C'est la source de vérité que Claude lit avant chaque action. Sans lui, Claude improvise. Avec lui, Claude exécute ta vision.

Qu'est-ce que CLAUDE.md ?

Un fichier CLAUDE.md placé à la racine du projet. Claude Code le lit automatiquement au démarrage et l'utilise comme contexte permanent pour toute la session.

C'est ton brief de projet vivant : vision, architecture, règles, stack, conventions, interdictions.

Règle d'or

Tout ce que tu ne veux pas répéter à Claude à chaque session → mets-le dans le CLAUDE.md. C'est ta mémoire externalisée.

Structure d'un CLAUDE.md efficace
CLAUDE.md
# NOM DU PROJET — CLAUDE.md

## VISION
[Produit en 2-3 phrases. Pour qui. Quel problème.]

## STACK
Frontend  : React + TypeScript + Vite
Styling   : Tailwind CSS + CSS Variables
Backend   : Supabase (Auth, DB, Storage)
Hosting   : Vercel

## RÈGLES DE CODE
- TypeScript strict — jamais de `any`
- Composants fonctionnels uniquement
- Imports absolus depuis src/

## DESIGN
- Fond : #080909 — jamais blanc
- Font : Geist (UI) + Instrument Serif (titres hero)

## INTERDICTIONS
- JAMAIS Anthropic API côté client
- JAMAIS committer .env.local
- JAMAIS de gradients colorés

Les Skills —

Les Skills sont des fichiers d'instructions spécialisées chargés à la demande. Des "modes experts" que tu actives selon la tâche : design, code review, copywriting...

Qu'est-ce qu'un Skill ?

Un Skill = un fichier SKILL.md dans un dossier dédié. Il contient des instructions précises pour un rôle ou une tâche. Tu l'invoques avec /skill nom dans Claude Code.

Analogie

CLAUDE.md = brief permanent. Skills = experts que tu appelles selon le besoin : le graphiste, le dev, le marketeur — tu choisis.

problème + ligne + solution proposée.

Chapitre 09⏱ 10 min

Maîtriser
le prompting

Un bon prompt fait la différence entre une réponse générique et une réponse exploitable. La méthode Buildrs pour formuler des instructions qui produisent exactement ce que tu veux.

La formule RCTF
  • Rôle : qui est Claude dans ce contexte ? ("Tu es un expert en UX design...")
  • Contexte : quelle est la situation ? ("Je construis une app mobile pour...")
  • Tâche : qu'est-ce que tu veux exactement ? ("Génère 5 noms...")
  • Format : comment doit être la réponse ? ("En JSON / liste / 3 phrases max")
Avant / après
❌ Mauvais prompt
Aide-moi avec mon app
✅ Bon prompt
Tu es un expert en création de Micro-SaaS.
Je construis une app de gestion de factures
pour les freelances français.

Génère 3 noms courts et mémorables.
Contraintes : max 2 syllabes, pas de mots anglais,
évocateur de rapidité ou de clarté.

Format : liste numérotée + 1 ligne d'explication.
Techniques avancées
  • Zero-shot : demande directement sans exemple. Efficace pour des tâches que Claude maîtrise.
  • Few-shot : donne 2-3 exemples avant ta question. Réduit les erreurs de 40%+.
  • Chain of thought : Réfléchis étape par étape avant de répondre. Claude arrive à de meilleures conclusions.
  • Contraintes négatives : Ne propose pas X, Y ou Z. Oriente la réponse par l'exclusion.
Itérer — le vrai apprentissage
  • Version 1 : prompt basique → évalue la réponse
  • Identifie ce qui manque : trop vague ? Mauvais format ? Mauvais ton ?
  • Affine : ajoute les contraintes manquantes
  • Sauvegarde le meilleur : dans ton fichier prompts.md
Technique Buildrs

Après un bon prompt, dis à Claude : Analyse ce prompt. Qu'est-ce qui le rend efficace ? Comment l'améliorer encore ? C'est ta formation continue gratuite.

✦ Action concrète
Améliore un prompt existant

    Prends une question posée à Claude dont la réponse était insuffisante.
  • 2

  • 3
    Envoie les 2 versions et demande : Laquelle est plus efficace et pourquoi ?
  • 4
    Sauvegarde le meilleur dans un fichier prompts.md dans ton projet.
Chapitre 10⏱ 10 min

MCP — connecter
Claude au monde

Par défaut, Claude est isolé. Les MCP (Model Context Protocol) sont des ponts entre Claude et tes outils : Supabase, GitHub, Notion, Stripe...

MCP — qu'est-ce que c'est ?

MCP est un standard ouvert créé par Anthropic. Il permet à Claude de se connecter à des services externes via des "serveurs MCP" — de petits programmes qui exposent des outils à Claude.

Exemple : installe le MCP Supabase, et Claude peut créer des tables, faire des requêtes SQL, gérer les migrations — directement depuis le terminal.

MCP essentiels pour Buildrs

mcp-supabaseTables, requêtes SQL, Auth, migrations
mcp-githubIssues, PRs, lire/écrire fichiers repos
mcp-notionCréer/modifier pages et bases de données
mcp-vercelDéployer, logs, gérer projets
mcp-googleGmail, Calendar, Drive

Chapitre 11⏱ 12 min

Tes premiers
workflows IA

Un agent seul, c'est bien. Une chaîne d'agents coordonnés, c'est une machine. Ce chapitre t'apprend à orchestrer plusieurs outils pour créer des résultats répétables sans intervention humaine.

Qu'est-ce qu'un workflow IA ?

Un workflow IA = une séquence d'étapes automatisées où des agents IA transforment un input en output, sans intervention humaine à chaque étape.

Exemple : Email entrant → Claude analyse → Classifie → Crée task Notion → Notifie Slack. Tu n'interviens qu'en cas d'exception.

Les 4 types de workflows Buildrs
  • Veille : surveille des sources → Claude résume → envoi quotidien
  • Contenu : idée → Claude rédige → format LinkedIn/blog → scheduling
  • Support : demande client → Claude classifie et répond → escalade si besoin
  • Build : brief → Claude Code crée features → tests → déploiement Vercel
n8n + Claude — le combo puissant

n8n est l'outil d'automatisation open-source utilisé par Buildrs. Il se connecte à 400+ apps et peut appeler Claude via l'API Anthropic.

Workflow n8n — Résumé quotidien
1. TRIGGER : Schedule (7h chaque matin)
   ↓
2. HTTP Request → RSS feeds (tech, SaaS, IA)
   ↓
3. Claude API → "Résume ces articles en 5 points
   clés actionnables pour un builder SaaS"
   ↓
4. Format → Markdown structuré
   ↓
5. Send → Email + Notion + Slack
Règles d'un bon workflow
  • Un seul input clair : chaque workflow commence par une source définie
  • Gestion d'erreur : que se passe-t-il si Claude échoue ? Si l'API timeout ?
  • Output testable : tu dois pouvoir vérifier que le résultat est correct
  • Human in the loop : garde une validation humaine sur les actions irréversibles
✦ Action concrète
Dessine ton premier workflow
  • 1
    Identifie une tâche répétée chaque semaine : rapport, résumé, tri d'emails, publication...
  • 2
    Dessine sur papier : Input → Étape 1 → Étape 2 → Output. Identifie où Claude intervient.
  • 3
    Dans Claude.ai : Comment automatiser ce workflow avec n8n et Claude ? Donne la structure step-by-step.
  • 4
    Teste manuellement une fois avant d'automatiser. Valide que le résultat est bon.
